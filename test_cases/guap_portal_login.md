# Тест-кейсы: Личный кабинет студента ГУАП

**Фича:** Вход в личный кабинет на student.guap.ru  
**Автор:** Ситников С. — студент гр. Z3420, ГУАП  
**Статус:** учебный проект, тестирование вручную

---

## TC-01 — Успешный вход по логину и паролю

**Приоритет:** Critical  
**Тип:** Позитивный

**Предусловия:** Студент зачислен, учётная запись активирована деканатом.

**Шаги:**
1. Открыть student.guap.ru
2. Ввести корректный логин (номер студенческого)
3. Ввести корректный пароль
4. Нажать «Войти»

**Ожидаемый результат:** Открывается личный кабинет с ФИО студента, группой и семестром.

---

## TC-02 — Вход с неверным паролем

**Приоритет:** High  
**Тип:** Негативный

**Шаги:**
1. Открыть страницу входа
2. Ввести корректный логин
3. Ввести неверный пароль: `wrongpassword`
4. Нажать «Войти»

**Ожидаемый результат:** Ошибка «Неверный логин или пароль». Страница входа остаётся открытой. Пользователь не авторизован.

---

## TC-03 — Вход с пустыми полями

**Приоритет:** High  
**Тип:** Негативный

**Шаги:**
1. Открыть страницу входа
2. Оставить поля логина и пароля пустыми
3. Нажать «Войти»

**Ожидаемый результат:** Форма не отправляется. Поля подсвечиваются как обязательные.

---

## TC-04 — Поле пароля скрывает ввод

**Приоритет:** Medium  
**Тип:** Безопасность

**Шаги:**
1. Открыть страницу входа
2. Кликнуть на поле пароля
3. Ввести любой текст

**Ожидаемый результат:** Символы отображаются точками/звёздочками. Пароль не виден в открытом виде.

---

## TC-05 — Восстановление пароля (актуально для первокурсников)

**Приоритет:** Medium  
**Тип:** Функциональный

**Шаги:**
1. Открыть страницу входа
2. Нажать «Забыли пароль?»

**Ожидаемый результат:** Открывается форма восстановления с полем для студенческого e-mail.

---

## TC-06 — Сессия сохраняется после перезагрузки вкладки

**Приоритет:** Medium  
**Тип:** Функциональный

**Шаги:**
1. Выполнить успешный вход (TC-01)
2. Нажать F5 (обновить страницу)

**Ожидаемый результат:** Пользователь остаётся авторизованным. Повторный ввод пароля не требуется.

---

## TC-07 — Выход из личного кабинета

**Приоритет:** Medium  
**Тип:** Функциональный

**Шаги:**
1. Выполнить вход (TC-01)
2. Нажать «Выйти»

**Ожидаемый результат:** Сессия завершена. Открывается страница входа. При нажатии «Назад» в браузере — кабинет не открывается.

## ✅ Smoke-чеклист: Вход в личный кабинет

Запускать после каждого деплоя или обновления портала.

- [ ] Страница входа открывается без ошибок
- [ ] Поле «Логин» доступно для ввода
- [ ] Поле «Пароль» доступно для ввода и скрывает символы
- [ ] Кнопка «Войти» кликабельна
- [ ] Вход с корректными данными → личный кабинет открывается
- [ ] Вход с неверным паролем → сообщение об ошибке

---

## ✅ Регрессионный чеклист: Вход в личный кабинет

Полная проверка перед началом семестра / после технических работ.

**Позитивные сценарии:**
- [ ] TC-01: Вход с верными данными
- [ ] TC-06: Сессия сохраняется при перезагрузке
- [ ] Вход работает в Chrome, Firefox, Safari (мобильный браузер)

**Негативные сценарии:**
- [ ] TC-02: Неверный пароль → ошибка, не авторизован
- [ ] TC-03: Пустые поля → форма не отправляется
- [ ] Несуществующий логин → ошибка (такая же, как при неверном пароле — не раскрываем, есть ли логин в системе)
- [ ] SQL-инъекция в поле логина (`' OR '1'='1`) — не должно приводить к входу или ошибке сервера

**Безопасность:**
- [ ] TC-04: Пароль скрыт при вводе
- [ ] Пароль не отображается в URL и не сохраняется в истории браузера
- [ ] После 5 неверных попыток — блокировка или капча

**Граничные значения:**
- [ ] Логин из 1 символа — ошибка валидации
- [ ] Логин из 255 символов — не должен вызывать ошибку сервера
- [ ] Пробелы в начале/конце логина — обрезаются автоматически (trim)
