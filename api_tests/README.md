# API Tests - Расписание ГУАП (учебный проект)

Идея: представил, что тестирую бэкенд системы расписания ГУАП.  
Для отработки навыков использую публичный тестовый API JSONPlaceholder.

## Маппинг endpoints

| Endpoint | В контексте ГУАП |
|----------|-----------------|
| `GET /users` | Список преподавателей кафедры |
| `GET /users/{id}` | Карточка преподавателя |
| `GET /posts` | Расписание занятий на семестр |
| `GET /posts?userId=N` | Расписание конкретного преподавателя |
| `GET /todos` | Домашние задания |
| `POST /todos` | Добавить новое задание |
| `PATCH /todos/{id}` | Отметить задание выполненным |

## Что отработал

- Базовые HTTP-запросы через `requests`
- Проверка статус-кодов (200, 201, 404)
- Проверка структуры JSON-ответа
- Фильтрация данных через query-параметры
- Параметризация тестов через `@pytest.mark.parametrize`
- Session fixture для переиспользования соединения

## Запуск

```bash
pip install pytest requests
pytest api_tests/ -v
```
